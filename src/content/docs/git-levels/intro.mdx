Git nivåer er et konsept som er fundamentalt for å forstå hvordan Git sporer og håndterer filene dine. Disse nivåene er kritiske å forstå for å kunne samarbeide nøyaktig og effektivt i prosjekter.

Når du endrer, oppretter eller manipulerer filene i et Git-prosjekt, beveger de seg gjennom fire "stages" som bestemmer hvordan Git skal håndtere dem. La oss gå gjennom disse.

## De fire Git nivåene
- Untracked (Usporet): Dette er den opprinnelige statusen for nye filer. Git er foreløpig ikke klar over dem og vil ikke ta vare på endringer du gjør. Tenk på dette nivået som "utenfor Git sin radar".
- Staged (Klargjort): Når du klargjør en fil, forteller du Git å begynne å følge den. Filen er klar for å bli en del av det neste "snapshot" eller bilde av prosjektet ditt.
- Committed (Bekreftet): Etter å ha bekreftet, er filen sikkerhetskopiert i din lokale Git-database. Her ligger endringene dine som en del av prosjektets historie.
- Pushed (Pushet): Å 'pushe' betyr å sende dine lokale bekreftede endringer opp til en ekstern server, som GitHub, hvor andre også kan se og samarbeide på prosjektet.

Ved å navigere disse nivåene blir du tryggere i å kontrollere historikken og arbeidsflyten i dine prosjekter.
