## Remote Repository: Lagre repository trygt og delbart
Det fjerde og siste nivået i Git er remote repository. Dette er en versjonskontrollserver - som GitHub, hvor dine commits blir lagret eksternt og trygt. Dette beskytter ditt arbeid mot lokale dataproblemer, og gjør prosjektet ditt tilgjengelig for samarbeid, enten med kollegaer, medelever eller opensource-fellesskapet!

## Hvordan flytter vi endringer mellom lokalt og remote repository?
For å overføre dine lokale commits til det eksterne repositoryet bruker vi git push-kommandoen. Dette gir deg muligheten til å dele dine commits og oppdateringer med verden. Eller bare med deg selv som en backup. På motsatt side, dersom du trenger å hente oppdateringer fra det eksterne repositoryet til ditt lokale arbeidsområde, kan du bruke git pull-kommandoen. Dette er nyttig for å få de nye commits andre har lagt til, eller for å synkronisere arbeidet ditt mellom forskjellige datamaskiner.

## Et Praktisk Eksempel
Forestill deg at du har fullført et omfattende skolearbeid og er klar til å dele det med gruppelæreren din. Du har flere commits i ditt lokale repository. Nå bruker du kommandoen git push for å overføre commits til GitHub. Etter en stund, finner du ut om en liten feil du ønsker å fikse. Om du ikke er på den samme maskinen, kan du enkelt bruke git pull for å laste ned det nyeste arbeidet ditt, gjøre endringene, og deretter igjen git push for å oppdatere prosjektet på GitHub.
